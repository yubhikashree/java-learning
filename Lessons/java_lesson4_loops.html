Lesson 4 of Phase 0

Loops — Making Java repeat itself

Loops are one of the most powerful tools in programming. Without them, printing numbers 1 to 1000 would mean writing 1000 lines of code. With a loop — three lines. Every algorithm, every data structure, every AI model uses loops at its core.

Part 1 — The for loop

Anatomy of a Java for loop

for (int i = 0; i < 5; i++) { System.out.println(i); }
PART 1 — Start
int i = 0
Runs once. Creates the counter variable i, starts at 0.
PART 2 — Condition
i < 5
Checked before every iteration. Loop runs while this is true.
PART 3 — Step
i++
Runs after every iteration. i++ means add 1 to i each time.
1
i++ is shorthand for i = i + 1. You'll also see i-- (subtract 1), i += 2 (add 2), i *= 2 (double each time).
2
Output of the loop above: 0 1 2 3 4 — five numbers, starting at 0, stopping before 5. Sound familiar? Same logic as Python's range(5).
💡 To print 1 to 5 instead of 0 to 4: start at i = 1 and use i <= 5. The condition controls everything.
Part 2 — Loop variations

Different ways to control a for loop

Count up: 1 to 10
for (int i = 1; i <= 10; i++) { System.out.println(i); }
Count down: 10 to 1
for (int i = 10; i >= 1; i--) { System.out.println(i); }
Even numbers: 2 to 10
for (int i = 2; i <= 10; i += 2) { System.out.println(i); }
Multiples of 5
for (int i = 5; i <= 50; i += 5) { System.out.println(i); }
Part 3 — while loop

When you don't know how many times to loop

Use a for loop when you know the exact number of iterations. Use a while loop when you loop until a condition changes.

int i = 1; while (i <= 5) { System.out.println(i); i++; // ← never forget this — without it, infinite loop }
1
The while loop checks its condition first. If it's false from the start, the loop body never runs at all.
2
Always make sure something inside the loop moves it toward the exit condition. Forgetting i++ creates an infinite loop — your program freezes forever.
Good use of while — unknown iterations
Keep asking for input until the user types a valid mark (0–100). You don't know how many wrong attempts they'll make — so while is the right choice.
Good use of for — known iterations
Print a multiplication table for 5 (5×1 to 5×10). You know exactly 10 iterations — so for is the right choice.
Part 4 — Loops + conditionals combined

This is where real programs begin

Combining loops with if-else lets you process every number in a range and make decisions about each one. This pattern is the foundation of almost every algorithm you'll ever write.

// Print whether each number 1-10 is even or odd for (int i = 1; i <= 10; i++) { if (i % 2 == 0) { System.out.println(i + " is even"); } else { System.out.println(i + " is odd"); } }
💡 Remember modulus? i % 2 == 0 means "i divides evenly by 2 — no remainder". That's the standard even/odd check used in every language, every interview, every algorithm.
Part 5 — break and continue

Controlling loop flow

break — exit the loop immediately
for (int i = 1; i <= 10; i++) { if (i == 5) { break; } System.out.println(i); } // prints: 1 2 3 4
continue — skip this iteration only
for (int i = 1; i <= 5; i++) { if (i == 3) { continue; } System.out.println(i); } // prints: 1 2 4 5
1
break exits the entire loop. continue skips only the current iteration and moves to the next one. Both are tools — use them deliberately, not by default.
