Lesson 7 of Phase 0

Strings — Working with text in Java

String is the most used class in all of Java programming. Every user input, every file name, every message — it's all Strings. Mastering String methods makes you dramatically faster at solving problems.

Part 1 — Strings are character arrays

What a String actually looks like in memory

A String is internally a sequence of characters, each at an index — exactly like an array.

Y

u
1

String name = "Yubhi"; System.out.println(name.length()); // 5 System.out.println(name.charAt(0)); // Y System.out.println(name.charAt(4)); // i System.out.println(name.charAt(name.length()-1)); // i — last character
1
charAt(index) returns the character at that position. Returns a char type — a single character in single quotes like 'Y'.
2
name.length() - 1 always gives you the last character's index — same logic as arrays.
Part 2 — Essential String methods

The methods you'll use in every program

String s = " Hello, World! "; // Case System.out.println(s.toUpperCase()); // " HELLO, WORLD! " System.out.println(s.toLowerCase()); // " hello, world! " // Trimming whitespace System.out.println(s.trim()); // "Hello, World!" // Searching System.out.println(s.contains("World")); // true System.out.println(s.indexOf("World")); // 9 — position where it starts System.out.println(s.startsWith(" He")); // true System.out.println(s.endsWith("! ")); // true // Extracting System.out.println(s.substring(2, 7)); // "Hello" System.out.println(s.trim().substring(7));// ", World!" // Replacing System.out.println(s.replace("World", "Java")); // " Hello, Java! " // Splitting String csv = "Yubhi,Arun,Priya"; String[] names = csv.split(","); System.out.println(names[0]); // Yubhi System.out.println(names[1]); // Arun System.out.println(names[2]); // Priya
1
substring(start, end) extracts from start up to but NOT including end. Same exclusive upper bound rule as range() in Python.
2
split(",") breaks a String into an array using a delimiter. This is how you process CSV data — something you'll do constantly in data science.
3
String methods never modify the original String — they always return a new one. Strings in Java are immutable — once created, they cannot be changed.
Part 3 — String comparison

The most important String rule in Java

🚨 NEVER use == to compare Strings. Always use .equals(). This is one of the most common bugs in Java and it appears in every fresher interview.
String a = "hello"; String b = "hello"; // WRONG — compares memory addresses, not content if (a == b) // may give unexpected results // CORRECT — compares actual content if (a.equals(b)) // true if (a.equalsIgnoreCase(b)) // true — ignores case difference
1
== checks if two variables point to the exact same object in memory. .equals() checks if the content is the same. For Strings, you almost always want content comparison.
2
equalsIgnoreCase() is useful for user input — if someone types "YES", "yes", or "Yes", they all mean the same thing.
Part 4 — Looping through a String

Processing character by character

String name = "Yubhi"; // Count vowels int vowelCount = 0; for (int i = 0; i < name.length(); i++) { char c = name.charAt(i); if (c=='a'||c=='e'||c=='i'||c=='o'||c=='u'|| c=='A'||c=='E'||c=='I'||c=='O'||c=='U') { vowelCount++; } } System.out.println("Vowels in " + name + ": " + vowelCount); // 2
1
char c = name.charAt(i) — note the type is char, not String. A char holds a single character and uses single quotes: 'a' not "a".
2
Character comparison uses == — not .equals(). The == rule only applies to String objects, not primitive char values.
Part 5 — Converting between types

String ↔ int ↔ double conversions

// String to int String numStr = "42"; int num = Integer.parseInt(numStr); // num = 42 System.out.println(num + 8); // 50 — actual addition // String to double String dblStr = "3.14"; double d = Double.parseDouble(dblStr); // d = 3.14 // int to String int age = 17; String ageStr = String.valueOf(age); // "17" String ageStr2 = "" + age; // "17" — quick trick // Useful: check if string is a number before parsing String input = "abc"; try { int val = Integer.parseInt(input); } catch (NumberFormatException e) { System.out.println("Not a valid number!"); }
1
Integer.parseInt() is used constantly — whenever you read a number as text and need to do math with it. If the String isn't a valid number, it throws NumberFormatException.
2
try-catch is your first look at exception handling — a full lesson coming soon. For now just know it catches errors gracefully instead of crashing.
