Lesson 5 of Phase 0

Methods — Packaging logic into reusable blocks

A method is a named block of code that does one specific job. Instead of writing the same logic repeatedly, you write it once, name it, and call it whenever needed. This is the foundation of every program larger than 50 lines.

Part 1 — void method: does a job, returns nothing

The simplest method — just does something

public class Main { // method defined beside main — not inside it static void greet(String name) { System.out.println("Hello, " + name + "!"); System.out.println("Welcome to Java."); } public static void main(String[] args) { greet("Yubhi"); // call 1 greet("Arun"); // call 2 — same method, different input greet("Priya"); // call 3 } }
1
void means this method gives nothing back. It just does its job and finishes. No return statement needed.
2
String name is the parameter — the input the method accepts. When you call greet("Yubhi"), the value "Yubhi" gets copied into name inside the method.
3
The same method runs three times with three different names. That's the power of methods — write once, use anywhere.
💡 DRY principle again: Without this method you'd write those two println statements three times — 6 lines. With the method — 3 lines. In a real program with 100 calls, this difference is enormous.
Part 2 — Methods that return a value

Methods that calculate and give back a result

static int add(int a, int b) { return a + b; // sends the result back to whoever called this } static double average(int a, int b, int c) { return (double)(a + b + c) / 3; } static String getGrade(int marks) { if (marks >= 90) return "A"; else if (marks >= 75) return "B"; else if (marks >= 60) return "C"; else return "F"; } public static void main(String[] args) { int sum = add(10, 20); // sum holds 30 double avg = average(80, 90, 85); // avg holds 85.0 String grade = getGrade(87); // grade holds "B" System.out.println("Sum: " + sum); System.out.println("Average: " + avg); System.out.println("Grade: " + grade); }
1
return sends a value back to wherever the method was called from. The returned value must match the declared return type — int method must return an int.
2
The returned value can be stored: int sum = add(10, 20) — or used directly: System.out.println(add(10, 20)).
3
Notice getGrade has multiple return statements — Java exits the method the moment it hits any return. Only one return ever executes per method call.
Part 3 — Method execution flow

What happens when a method is called

main() starts
→
calls add(10, 20)
→
add() runs
→
returns 30
→
main() continues

When Java hits a method call, it pauses the current method, jumps to the called method, runs it completely, then returns to exactly where it left off. This is called the call stack — one of the most important concepts in computer science.

Part 4 — Method overloading

Same name, different parameters

Java allows multiple methods with the same name as long as their parameters are different. This is called overloading.

static int add(int a, int b) { return a + b; } static double add(double a, double b) { return a + b; } static int add(int a, int b, int c) { return a + b + c; } public static void main(String[] args) { System.out.println(add(2, 3)); // calls first — prints 5 System.out.println(add(2.5, 3.5)); // calls second — prints 6.0 System.out.println(add(1, 2, 3)); // calls third — prints 6 }
1
Java picks the right method automatically based on what arguments you pass. This is called compile-time polymorphism — a term you'll hear in every Java interview.
2
You've already seen overloading without knowing it — System.out.println() works with int, double, String, boolean — that's because println is overloaded for every type.
Part 5 — Complete example: Grade Report

Your Exercise 3 rewritten with methods

import java.util.Scanner; public class Main { static double calcAverage(int a, int b, int c) { return (double)(a + b + c) / 3; } static String calcGrade(double avg) { if (avg >= 90) return "A"; else if (avg >= 75) return "B"; else if (avg >= 60) return "C"; else if (avg >= 40) return "D"; else return "F"; } static String calcResult(double avg) { if (avg >= 40) return "Pass"; else return "Fail"; } static void printReport(String name, int m, int p, int c) { double avg = calcAverage(m, p, c); String grade = calcGrade(avg); String result = calcResult(avg); System.out.println("--- Report Card ---"); System.out.println("Student : " + name); System.out.println("Total : " + (m+p+c) + "/300"); System.out.println("Average : " + avg + "%"); System.out.println("Grade : " + grade); System.out.println("Result : " + result); } public static void main(String[] args) { Scanner sc = new Scanner(System.in); System.out.print("Name: "); String name = sc.nextLine(); System.out.print("Maths: "); int m = sc.nextInt(); System.out.print("Physics: "); int p = sc.nextInt(); System.out.print("Chemistry: "); int c = sc.nextInt(); printReport(name, m, p, c); } }

Compare this to your original Exercise 3. Same program — but now every job has its own named method. main() only coordinates. Each method does one thing. This is how real software is written.
