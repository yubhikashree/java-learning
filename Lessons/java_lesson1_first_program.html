Java — your first program

Java is the language most used in coding interviews, backend systems, and Android apps. It's also what most Indian engineering colleges teach. Learning it now puts you 3–4 months ahead of your classmates.

Theory

How Java is different from Python

Python
print("Hello")

# Just run the file.
# Python figures
# out the rest.
Java
class Main {
  public static void main
  (String[] args) {
    System.out.println
    ("Hello");
  }
}

Java is more explicit — you have to tell it more things upfront. This feels verbose at first, but it makes large programs much safer and easier to debug. Every line means something.

Line by line

Anatomy of a Java program

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
1
public class Main — Every Java program lives inside a class. Think of a class as a container. The filename must match: this file is saved as Main.java.
2
public static void main(String[] args) — This is the entry point. When you run a Java program, Java looks for exactly this line and starts here. Every Java program must have it.
3
System.out.println(...) — This is Java's equivalent of Python's print(). The semicolon ; at the end is mandatory — Java uses it to know where one statement ends.
4
{ } — Curly braces replace Python's indentation for defining blocks. You still indent for readability, but the braces are what Java actually reads.
Key difference to remember

Semicolons and braces

In Python, a new line ends a statement. In Java, a semicolon ends a statement and curly braces define blocks. These two things cause 80% of beginner Java errors. The compiler will tell you exactly where you forgot them.

Python — newline ends statement
x = 5
y = 10
print(x + y)
Java — semicolon ends statement
int x = 5;
int y = 10;
System.out.println(x + y);
Notice something?

Java makes you declare types

In Python you write x = 5. In Java you write int x = 5; — you must tell Java that x holds an integer. This is called static typing. Java checks your types before the program even runs, catching a whole class of bugs early.

int age = 18; // whole numbers
double gpa = 9.2; // decimal numbers
String name = "Arjun"; // text
boolean passed = true; // true or false
