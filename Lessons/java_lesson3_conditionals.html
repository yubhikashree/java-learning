Lesson 3 of Phase 0

Conditionals — Making decisions in Java

Programs that can't make decisions are just calculators. Conditionals are what give your programs intelligence — the ability to behave differently based on input.

🔒 Golden rule you discovered yourself: Always use curly braces — even for single-line blocks. No exceptions.

Part 2 — if / else if / else

Full structure with curly braces

int marks = 85; if (marks >= 90) { System.out.println("Grade: A"); System.out.println("Outstanding!"); } else if (marks >= 75) { System.out.println("Grade: B"); System.out.println("Well done!"); } else if (marks >= 60) { System.out.println("Grade: C"); System.out.println("Keep improving."); } else { System.out.println("Grade: F"); System.out.println("Please revise."); }
1
Java checks conditions top to bottom and stops at the first true one. If marks is 85, it skips the first condition, matches the second, runs that block, and ignores everything else.
2
The else block at the end is a safety net — it runs only if none of the above conditions matched. Always include it to handle unexpected input.
Part 3 — Logical Operators in practice

Combining conditions with && and ||

int age = 20; boolean hasID = true; // AND — both conditions must be true if (age >= 18 && hasID == true) { System.out.println("Entry allowed"); } // OR — at least one must be true int x = 0, y = 5; if (x == 0 || y == 0) { System.out.println("At least one is zero"); } // NOT — flips the condition boolean isRaining = false; if (!isRaining) { System.out.println("Go for a walk!"); }
💡 Short-circuit evaluation: In &&, if the first condition is false, Java doesn't even check the second — it already knows the result is false. In ||, if the first is true, Java skips the second. This matters in interviews.
Part 4 — switch statement

When you have many exact values to check

int day = 3; switch (day) { case 1: System.out.println("Monday"); break; case 2: System.out.println("Tuesday"); break; case 3: System.out.println("Wednesday"); break; default: System.out.println("Other day"); }
1
break is mandatory after each case. Without it, Java "falls through" to the next case and runs it too — a very common bug called fall-through.
2
default is the switch equivalent of else — runs when no case matches.
3
Use switch when checking one variable against many exact values. Use if-else when checking ranges like >= 90.
