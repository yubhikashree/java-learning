Lesson 6 of Phase 0

Arrays — Storing multiple values in one variable

An array is a fixed-size, ordered collection of values of the same type. Instead of 100 separate variables for 100 students, you have one array with 100 slots. Arrays are the foundation of every data structure you'll ever learn.

Part 1 — Creating and visualising arrays

What an array looks like in memory

int[] marks = {85, 90, 78, 92, 88};

78

Five values, five slots, indices 0 through 4. The array has a fixed size of 5 — you cannot add a 6th element later. That's the key difference from Python lists.

// Three ways to create arrays // Way 1 — declare and initialise together int[] marks = {85, 90, 78, 92, 88}; // Way 2 — declare size first, fill later int[] marks = new int[5]; marks[0] = 85; marks[1] = 90; marks[2] = 78; // Way 3 — declare, then initialise separately int[] marks; marks = new int[]{85, 90, 78, 92, 88};
1
int[] means "array of integers". The square brackets tell Java this is an array, not a single int.
2
new int[5] creates 5 empty slots, all initialised to 0 by default. For String arrays, default is null. For boolean arrays, default is false.
⚠️ Fixed size: Once you create new int[5], it has exactly 5 slots forever. You cannot resize it. If you need a resizable collection, use ArrayList — coming in Lesson 7.
Part 2 — Accessing and modifying elements

Reading and writing array values

int[] marks = {85, 90, 78, 92, 88}; // Reading System.out.println(marks[0]); // 85 — first element System.out.println(marks[4]); // 88 — last element System.out.println(marks.length); // 5 — size of array // Modifying marks[2] = 95; // changes 78 to 95 System.out.println(marks[2]); // now prints 95
🚨 ArrayIndexOutOfBoundsException — the most common array error. If your array has 5 elements (indices 0–4) and you try marks[5], Java throws this error and crashes. Always remember: last valid index = length - 1.
💡 marks.length — not marks.length(). No parentheses. Length is a property, not a method. This catches everyone at least once.
Part 3 — Looping through arrays

Arrays and loops — inseparable partners

int[] marks = {85, 90, 78, 92, 88}; // Standard for loop for (int i = 0; i < marks.length; i++) { System.out.println("Student " + (i+1) + ": " + marks[i]); } // Enhanced for loop (for-each) — cleaner when you don't need index for (int mark : marks) { System.out.println(mark); }
1
Always use i < marks.length as your condition — not i <= marks.length. The last valid index is length - 1, so < is correct.
2
The enhanced for loop for (int mark : marks) reads as "for each mark in marks". Use it when you only need the value, not the position. Python equivalent: for mark in marks.
Part 4 — Common array operations

Sum, average, max, min — the core four

int[] marks = {85, 90, 78, 92, 88}; // Sum int sum = 0; for (int mark : marks) { sum += mark; // shorthand for sum = sum + mark } System.out.println("Sum: " + sum); // 433 // Average double avg = (double) sum / marks.length; System.out.println("Average: " + avg); // 86.6 // Maximum int max = marks[0]; // assume first is biggest for (int i = 1; i < marks.length; i++) { if (marks[i] > max) { max = marks[i]; } } System.out.println("Max: " + max); // 92 // Minimum int min = marks[0]; // assume first is smallest for (int i = 1; i < marks.length; i++) { if (marks[i] < min) { min = marks[i]; } } System.out.println("Min: " + min); // 78
1
Always initialise max and min to marks[0] — the first element. Never initialise to 0, because if all marks are negative, 0 would be wrong.
2
sum += mark is shorthand for sum = sum + mark. Similarly -=, *=, /=. You'll use these constantly.
Part 5 — Taking array input from user

Building an array dynamically

import java.util.Scanner; public class Main { public static void main(String[] args) { Scanner sc = new Scanner(System.in); System.out.print("How many students? "); int n = sc.nextInt(); int[] marks = new int[n]; // size decided by user for (int i = 0; i < n; i++) { System.out.print("Enter mark for student " + (i+1) + ": "); marks[i] = sc.nextInt(); } System.out.println("You entered: "); for (int mark : marks) { System.out.println(mark); } } }
1
The array size new int[n] is determined at runtime based on user input. This is dynamic sizing — the size is fixed once created, but chosen by the user.
